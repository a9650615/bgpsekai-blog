# `src/lib/baha-danmu/`

Pure-TypeScript port of the Bahamut Anime danmu ‚Üí Aegisub `.ass` converter that
used to live as an inline `<script>` on the Ghost page `/baha-danmu-to-ass/`.

## What it does

Given a Bahamut anime video SN (or a full `https://ani.gamer.com.tw/animeVideo.php?sn=<n>` URL),
the library:

1. fetches the danmaku list from the upstream API (via a public CORS proxy),
2. computes vertical stack positions and colour styles, and
3. emits an Aegisub-compatible `.ass` subtitle file using the legacy Ghost
   page's exact template (PlayResX 1920, PlayResY 1080, styles
   `Main` / `Story` / `ED_*` / `Annotation_*` / `Staff_*` / `Draw`, plus a
   per-color generated `Style:` line).

The UI lives in `src/components/BahaDanmuTool.astro` (with
`BahaDanmuTool.client.ts` for hydration) and is mounted at
`src/pages/baha-danmu-to-ass.astro`.

## Inputs / outputs

```ts
import { generateAss, fetchDanmu, extractSn } from './index.js';

const sn = extractSn('https://ani.gamer.com.tw/animeVideo.php?sn=15577');
const items = await fetchDanmu(sn);                     // DanmuItem[]
const ass = generateAss(items, { offset: -2 });         // string  (full .ass file)
```

`GenerateOptions` lets you override:

- `offset`         ‚Äî shift in **seconds** (positive or negative).
- `screenX/Y`      ‚Äî `PlayResX`/`PlayResY` (defaults 1920/1080).
- `fontSize`       ‚Äî used by `Main` and per-color generated styles.
- `danmutime`      ‚Äî duration appended to scrolling-danmu end time.
- `danmutimeFixed` ‚Äî duration for pinned danmu.
- `spaceHeight`, `centerOffsetY` ‚Äî vertical layout knobs.

## Limitations and caveats (please read)

- **`corsproxy.io` is a third-party** general-purpose CORS proxy. If it goes
  down or rate-limits, the tool will return a 4xx/5xx ‚Äî the user-facing UI
  surfaces the error message.
- The upstream **Bahamut API now restricts non-logged-in users** to a partial
  danmu set (some recent shows return an empty array). The UI surfaces that
  case as `ÂΩàÂπïÁÇ∫Á©∫,ÁÑ°Ê≥ïÁî¢ÁîüÂ≠óÂπï`.
- `dan.time` is in **deciseconds** (1/10 sec), not centiseconds. The original
  Ghost-page implementation derives hours as `t / 36000`, etc. The library
  follows that convention exactly.
- Several arithmetic quirks from the original implementation are intentionally
  preserved so the generated `.ass` byte-diffs cleanly against the legacy tool:
  - `dan.text.match(/[AZaz09]/g)` is a literal char class (six chars: `A Z a z 0 9`),
    *not* the alphanumeric range. We faithfully port that.
  - End-second can exceed 60 (e.g. `0:0:58 ‚Üí 0:0:68`) because `time[2] +
    danmutime` is rendered without carrying into the minute.
  - Minute / second are emitted **without** zero-padding (the upstream used
    `parseInt(padStart(2, ‚Ä¶))` which round-trips back to a single digit).
  - Per-color Style names embed a literal `#` (e.g. `Style: #FF0000,‚Ä¶`). Aegisub
    accepts this.
- The single-digit decisecond suffix (`.D`) is a one-character precision; this
  matches the legacy output and Aegisub renders it without complaint.

## ASS template provenance

The `[Script Info]` and `[V4+ Styles]` block (including the `ED_JP`,
`ED_SC`, `ED_TC`, `Main`, `Story`, `Annotation_*`, `Staff_*`, `Draw` styles
and the trailing whitespace on header lines like `Title: `) was lifted
verbatim from the inline `<script>` on the Ghost page
`https://blog.bgpsekai.club/baha-danmu-to-ass/`. See `template.ts` for the
parameterised version.

## How to extend with a new style

To add a Style line, edit `template.ts` and append a new line in
`buildTemplate`. Then either reference it from `pickStyle` (color routing) or
have the caller embed the style name directly in their danmu data flow.
